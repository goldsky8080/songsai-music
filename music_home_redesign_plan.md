# Music Home Redesign Plan

## 목표

`D:\music` 의 홈 화면을 현재의 기능 중심 업무형 UI에서,
브랜드 감도와 움직임이 살아 있는 **스튜디오형 랜딩 + 생성 콘솔** 경험으로 끌어올린다.

레퍼런스 방향:

- 강한 첫인상
- 깊이감 있는 레이어
- 큰 타이포그래피
- 부드러운 등장 모션
- 카드와 패널의 고급스러운 질감

단, 현재 서비스의 핵심 기능 흐름은 유지한다.

- 로그인
- 크레딧 확인
- 음악 생성
- 최근 생성 음악 확인
- 듣기 / 다운로드 / 비디오 생성

## 현재 구조 요약

현재 홈은 사실상 [auth-panel.tsx](D:/music/components/auth-panel.tsx) 하나에
아래 영역이 대부분 모여 있다.

- 로그인 화면
- 사용자/크레딧 요약
- 음악 생성 입력
- 최근 생성 음악

즉 기능은 충분하지만, 화면 밀도가 높고 “관리형 폼” 느낌이 강하다.

## 리디자인 방향

### 방향 A: Warm Futuristic Studio

현재 서비스의 크림/골드/브라운 계열을 유지하되,
더 미래적인 글로우와 큰 여백, 레이어드 배경을 넣는다.

핵심 요소:

- 큰 히어로 카피
- 미세하게 움직이는 배경 광원/오브젝트
- 반투명 카드 + 깊은 그림자
- 생성 영역을 메인 콘솔처럼 강조

장점:

- 현재 색감과 자연스럽게 이어짐
- “트로트 스튜디오” 정체성이 유지됨
- 기능 화면과 브랜드 랜딩 사이 균형이 좋음

### 방향 B: Editorial Luxury Interface

보다 정적인 편집디자인 감성으로,
타이포와 카드 배치를 크게 쓰고 여백으로 고급감을 만든다.

핵심 요소:

- 굵은 제목 + 세리프 강조
- 분명한 그리드
- 모션은 적고 질감 중심

장점:

- 구현 리스크가 낮음
- 현재 폰트 분위기와 잘 맞음

## 선택 방향

1차 구현은 **방향 A: Warm Futuristic Studio** 로 간다.

이유:

- 사용자 첫인상 개선 효과가 큼
- 레퍼런스 사이트의 “움직임 있는 브랜드 랜딩” 감성을 가장 잘 반영할 수 있음
- 현재 서비스의 골드/크림 톤을 버리지 않아도 됨

## 1차 구현 범위

1차는 아래 범위까지만 진행한다.

### 1. 홈 상단 히어로

- 한 줄 설명이 아니라 **브랜드형 도입부**로 변경
- 큰 제목
- 짧은 보조 설명
- 현재 크레딧/진행 상황을 한눈에 보이는 하이라이트 카드로 배치

### 2. 음악 생성 영역

- 지금의 생성 폼을 “메인 작업 콘솔”처럼 재배치
- 카드 크기와 여백 확대
- 섹션 간 위계 정리
- 버튼 강조 재설계

### 3. 최근 생성 음악

- 단순 리스트보다 쇼케이스형 카드로 재정리
- 상태 뱃지/듣기/다운로드 버튼 위계 개선
- 진행 중 항목이 더 잘 보이게 조정

## 2차 구현 범위

### 1. 마이크로 모션

- 페이지 첫 진입 시 패널 등장 애니메이션
- 카드 stagger
- 배경 광원 또는 blur shape 의 느린 움직임
- 버튼 hover 질감 강화

### 2. 전역 토큰 정리

- 색상 변수 확장
- 글로우/그림자 프리셋
- 카드 surface 체계화

### 3. 모바일 최적화

- 좁은 화면에서도 히어로 문구가 과하게 눌리지 않게 조정
- 생성 폼과 리스트가 세로 흐름에서 자연스럽게 이어지도록 재정리

## 3차 구현 범위

- `music-history`
- `credits`
- `admin`

까지 같은 디자인 언어로 확장

## 구현 순서

### Step 1

- 홈 래퍼([page.tsx](D:/music/app/page.tsx))에
  배경 레이어와 히어로 구조 도입

### Step 2

- [auth-panel.tsx](D:/music/components/auth-panel.tsx)
  상단 사용자 카드와 생성 카드 레이아웃 재정리

### Step 3

- 최근 생성 음악 리스트 카드 스타일 업그레이드

### Step 4

- 모션/등장 효과 추가

### Step 5

- 세부 간격, 반응형, 명암 대비 마감

## 구현 원칙

- 기능 흐름은 유지하고, 정보 구조와 시각 위계만 재설계한다
- 모바일 우선으로 봤을 때도 무너지지 않게 한다
- 과도한 AI 느낌보다 “의도된 브랜드 사이트”처럼 보이게 한다
- 기존 크레딧/생성/다운로드 로직은 손대지 않는다
